Dynamic Load Balancing for Distributed Barnes-Hut Simulations

Summary

We are implementing a dynamic load balancer for a distributed Barnes-Hut simulation running on a heterogeneous computing cluster. The system will use MPI and will also support execution across campus machines.

Background

Barnes-Hut is an N-body simulation algorithm that computes forces between bodies using hierarchical spatial partitioning. It reduces complexity compared to naive pairwise computation while maintaining accuracy.

Work assignment strategies include:

  • Static
  • Semi-static (well-suited for Barnes-Hut)
  • Dynamic

Semi-static approaches leverage slow-changing spatial distributions to minimize synchronization overhead.

Challenges

  • Non-uniform workload per body
  • Fine-grained and inefficient communication patterns
  • Dynamic redistribution due to movement of bodies
  • Highly heterogeneous hardware (GPUs vs CPUs, virtualized systems)
  • Variable system load due to shared campus machines

Goals & Deliverables

Plan

  • Implement Barnes-Hut with MPI
  • Build a dynamic load balancer
  • Optimize performance on a single machine
  • Deploy across multiple campus systems
  • Benchmark and analyze performance

Stretch Goals

  • Scale to more machines
  • Compare heterogeneous cluster configurations

Platform Choice

We use MPI for communication across distributed systems. Our environment intentionally includes heterogeneous and geographically distributed machines to evaluate robustness of the load balancing strategy.
